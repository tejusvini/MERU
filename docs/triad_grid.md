# The MERU Triad Grid

The triad grid is the structural backbone of MERU.

It organizes sequences on a **three-column lattice** that stabilizes pattern relationships.


L C R


---

## Grid Roles

### Center (C)

The center column forms the **primary structural spine**.

Properties:

- sequential ordering
- reference axis for patterns
- anchor for symmetry operations

---

### Left (L) and Right (R)

The lateral columns represent **structural relationships relative to the center**.

Common uses:

- mirror relationships
- contextual dependencies
- parallel pattern structures

---

## Example

Sequence:


A B C D E


Triad grid representation:


L C R
A
B C D
E


This layout allows MERU to detect structural properties such as:

- symmetry
- repetition
- mirrored relationships

---

## Structural Constraint

A key rule in MERU is:

**All lateral structures must resolve back to the center column.**

This prevents uncontrolled branching and keeps pattern growth stable.

---

## Why Three Columns?

Three columns provide the minimal structure needed to represent:

- linear sequences
- bilateral symmetry
- structural growth

A smaller structure cannot represent lateral relationships.  
A larger structure introduces unnecessary complexity.

The triad grid is therefore the smallest stable lattice for pattern analysis.
